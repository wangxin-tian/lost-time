# 常见性能优化方案

```bash
/**
 * 一个万能的优化思路, 关注整个系统上面
 */

let core_优化思路 = `
      1. 浏览器控制台中查看js加载情况 -> 
      2. 第三方依赖分析工具(webpack-bundle-analyzer) ->
      3. 外联加载(rel:proload,prefetch)对不需要首页不需要加载的信息进行低忧的加载
      4. 优先加载首页js，对其他的js内容使用延时加载或者动态加载(lazy(import()), ()=>{import()})
      5. 路由懒加载或者异步加载，对首页需要的js进行拆包分包加载
      6. 数据分析和处理的库可以在真正使用的使用再去加载(使用es6的动态加载)
      7. 日期的处理包有国际化，平时使用只需要加载中文就可以了
      8. 将用户不会访问的js拆分出来按需加载
      9. 利用es module中的动态加载，对涉及的功能模块按需加载
`;
```
